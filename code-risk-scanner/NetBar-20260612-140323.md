## 高优先级问题详情

### 问题 1: 更新包 sha256 字段已解码但未用于下载校验

**文件位置：** `Sources/NetBar/AppUpdater.swift:35`、`Sources/NetBar/AppUpdater.swift:529`

**问题代码：**
```swift
struct ReleaseManifest: Decodable, Equatable {
    let sha256: String
}

private func downloadWithProgress(asset: GitHubReleaseAsset) async throws -> URL
```

**上下文分析记录：**
- Step 1: 已读取 AppUpdater.swift 完整下载、解压、校验、安装链路。
- Step 2: `sha256` 来源是 `latest.json` manifest，但 `GitHubReleaseAsset` 和 `AvailableUpdate` 没有携带 expected checksum。
- Step 3: 当前防护包括 HTTP 状态、ZIP magic、Bundle 版本、codesign 验证。
- Step 4: 安装链路会替换当前 app；Release manifest 资产来自 GitHub。
- Step 5: 确认增强项：完整性字段存在但未闭环。若未来 manifest 是可信发布元数据，应下载后校验 ZIP hash。

**运行时/安全影响：**
- 不能发现下载资产和 manifest 声明不一致。
- codesign 验证不能替代发行方身份校验，尤其当前构建为 ad-hoc 签名时。

**修复建议：**
1. 在 `GitHubReleaseAsset` 或 `AvailableUpdate` 中携带 `expectedSHA256`。
2. `downloadWithProgress` 完成后计算 ZIP SHA-256。
3. 不匹配时阻止解压和安装，并给出明确错误。

### 问题 2: macOS 13 目标下 NSImage 跨并发任务触发 Sendable 可用性警告

**文件位置：** `Sources/NetBar/CustomCharacterImageProcessor.swift:456`

**问题代码：**
```swift
try await withThrowingTaskGroup(of: (Int, NSImage).self) { group in
    group.addTask {
        (index, try await self.pixelatedAsync(frame, scale: pixelation))
    }
}
```

**上下文分析记录：**
- Step 1: 已读取 CustomCharacterImageProcessor.swift 处理静态图、GIF、帧序列、像素化逻辑。
- Step 2: `NSImage` 从 UI/文件读取后进入 task group 和 detached task。
- Step 3: release build 明确产生 macOS 14 才有 NSImage Sendable conformance 的警告；项目最低支持 macOS 13。
- Step 4: 该路径用于自定义角色导入/再处理，属于用户可触发功能。
- Step 5: 确认问题/兼容性风险：当前能构建，但 Swift 并发检查趋严后可能变成错误，也可能掩盖 AppKit 对象跨线程使用风险。

**修复建议：**
1. 在进入并发任务前转成 `CGImage`、`Data`、或自定义 Sendable 像素缓冲。
2. 或将像素化处理串行化，避免跨 task 传递 `NSImage`。
3. 为 macOS 13 增加自定义角色处理的构建警告回归测试或 CI warning gate。

## 中优先级问题详情

### 问题 3: SystemPowerObserver 在 Sendable 闭包中直接修改 MainActor 属性

**文件位置：** `Sources/NetBar/SystemPowerObserver.swift:25`

release build 报告 `isScreenLocked` 在 Sendable closure 中直接赋值的 MainActor 隔离警告。前面的低电量通知已用 `Task { @MainActor in ... }` 包裹，屏幕休眠/唤醒两处也应保持一致。

### 问题 4: setPowerSaveMode 后采样定时器路径与启动路径不一致

**文件位置：** `Sources/NetBar/NetworkMonitor.swift:150`

启动时网络采样使用 `scheduleNextSample()` 的一次性 timer，并根据 `activityLevel.baseInterval` 和低电量状态动态安排下一次采样；`rescheduleTimers()` 改成固定 repeating timer，并且不管 `shouldSampleApplicationTraffic` 都创建应用流量 timer。建议统一为同一套调度函数，并只在详情窗口可见时创建应用流量 timer。

### 问题 5: 构建脚本静默吞掉 codesign 失败

**文件位置：** `Scripts/build-app.sh:41`

当前 `codesign ... || codesign ... || true` 会让签名失败也返回构建成功。对本地开发方便，但对 release/CI 不利。建议加入 `ALLOW_UNSIGNED_BUILD=1` 或 `CONFIGURATION=debug` 时才允许忽略，release 默认失败即退出。
